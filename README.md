# 日期倒计时 - 产品需求文档（PRD）

| 项目 | 说明 |
|------|------|
| **产品名称** | 日期倒计时 |
| **版本** | V1.0 |

## 2. 产品背景与目标

### 2.1 背景

用户在日常生活和工作中，经常需要关注某些重要日期的剩余时间，如节假日、生日、纪念日、项目截止日等。现有的日历工具缺乏直观的倒计时展示，用户难以一眼获知距离目标日期的精确剩余时间。

### 2.2 目标

打造一款**简洁美观、操作便捷**的日期倒计时工具，让用户能够：

- 快速创建多个自定义倒计时
- 实时查看距离目标日期的精确剩余时间（天、时、分、秒）
- 通过个性化的视觉主题区分不同事件
- 数据本地持久化，无需注册登录

## 7. 交互流程

### 7.1 新增倒计时

```
用户点击"+ 添加倒计时"
    → 弹出模态框（标题：添加倒计时）
    → 用户填写：事件名称、目标日期
    → 用户选择：emoji 图标、卡片主题色
    → 点击"确定"
        → 校验通过 → 保存到 localStorage → 刷新卡片列表 → 关闭弹窗
        → 校验失败 → 输入框抖动提示
    → 点击"取消"或遮罩 → 关闭弹窗，不保存
```

### 7.2 编辑倒计时

```
用户点击卡片"编辑"按钮
    → 弹出模态框（标题：编辑倒计时）
    → 自动回填当前事件的名称、日期、图标、主题
    → 用户修改内容
    → 点击"确定" → 更新数据 → 刷新列表
```

### 7.3 删除倒计时

```
用户点击卡片"删除"按钮
    → 弹出确认框："确定要删除这个倒计时吗？"
    → 点击"删除" → 移除数据 → 刷新列表
    → 点击"取消" → 关闭确认框
```

## 8. 数据结构

### 8.1 单条倒计时数据

```json
{
    "id": "cd_1713340800000_a1b2c3",
    "name": "元旦",
    "date": "2026-12-31T00:00:00",
    "emoji": "🎉",
    "theme": "pink"
}
```

### 8.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | string | 是 | 唯一标识，格式：`cd_时间戳_随机串` |
| name | string | 是 | 事件名称，最大 20 字符 |
| date | string | 是 | 目标日期，ISO 格式 `YYYY-MM-DDTHH:mm:ss` |
| emoji | string | 是 | emoji 图标 |
| theme | string | 是 | 主题色，可选值：`pink` `orange` `blue` `purple` `green` `red` |

### 8.3 存储方式

- **存储位置**：`localStorage`，key 为 `countdowns`
- **存储格式**：JSON 数组
- **容量预估**：单条数据约 150 字节，可存储数万条
